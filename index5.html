HTTP REQUEST HEADERS


  • HOST :-

    • It specifies domain name and port number of the server to which the request is being sent.


  • ACCEPT :-

    • It specifies which content type the client will accept in response.


  • ACCEPT-CHARSET :-

    • It indicates what character set are applicable for the response from the server.


  • ACCEPT-ENCODING :-

    • It indicates the content encoding the client can understand.

  • ACCEPT-LANGUAGE :-

    • It indicates the natural language the client can understand.

  • AUTHORIZATION :-

    • It is used to pass authentication credentials.

  • CACHE-CONTROL :-

    • No cache - It tells the server to revalidate the data before sending the response to make sure it's current.

    Max-age=(time) - It tells the server that client want data that's not older than mentioned time.


  • CONTENT-LENGTH :-

    • It indicates the length of request body in bytes.

  • CONTENT-TYPE :-

    • It indicates the content type of body of the request.

  • COOKIE :-

    • Sets a cookie.

  • REFERER :-

    • It contains the address of the previous web page from which a link to the currently requested page was followed.

  • USER-AGENT :-

    • The string that identifies the user agnt.
